# CAClaw Brand Inventory

This page lists where the CAClaw brand should appear in this repository.

## Primary CAClaw brand surfaces

These are the most important places where CAClaw should be visible:

- `README.md`
- `package.json`
- `openclaw.plugin.json`
- `index.ts`
- `assets/`
- user-facing docs in `docs/`

## Why this inventory matters

The brand should be consistent across:

- the GitHub landing page
- plugin metadata
- visual assets
- CA-facing documentation

## What should stay OpenClaw

The following should still clearly point to OpenClaw as the platform:

- installation commands such as `openclaw plugins install`
- OpenClaw documentation links
- any explanation of where the add-on runs
