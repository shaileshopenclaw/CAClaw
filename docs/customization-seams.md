# CAClaw Customization Seams

This page explains where future CAClaw work should go.

## Best places to add new CAClaw value

### Skills
Use skills for:

- checklists
- prompts
- workflow guidance
- recurring task structures

### Docs
Use docs for:

- user guidance
- process notes
- integration plans
- usage examples

### Assets
Use assets for:

- logo files
- preview images
- branded visuals for CAClaw

### Plugin runtime
Use plugin runtime files for:

- packaged CA functionality
- future commands or services
- logic that should ship with CAClaw itself

## What not to do

Do not turn CAClaw into a deep rewrite of OpenClaw.

The goal is:

- keep OpenClaw stable
- keep CAClaw additive
- keep future changes easy to review and maintain
