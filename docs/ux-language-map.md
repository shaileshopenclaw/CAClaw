# CAClaw UX Language Map

This page explains how CAClaw should talk to a Chartered Accountant.

## Plain-language rule

Lead with the business meaning first.
Use technical wording only when it helps the user complete the task.

## Examples

| Technical idea | Better CA-facing wording |
| --- | --- |
| session | conversation |
| prompt | working instruction |
| checklist asset | reusable checklist |
| workflow pack | CA workflow set |
| gateway | OpenClaw connection |

## Tone rule

CAClaw should sound:

- practical
- clear
- supportive
- professional

It should not sound:

- overly technical by default
- playful in a high-trust context
- like it is guaranteeing legal or tax correctness
