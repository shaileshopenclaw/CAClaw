# CA Bank Audit Workflow

Use this skill when the work is specifically a **bank audit**.

This skill is meant for Indian bank-audit support and should stay human-reviewed.

## What to do

1. Confirm the request is really bank-audit work.
2. Identify the bank-audit type:
   - statutory branch audit (SBA)
   - statutory central audit (SCA)
   - advances audit
   - NPA / IRACP / provisioning review
   - LFAR support
   - treasury / investments audit
   - deposits and liabilities review
   - RBI compliance review
3. Read `references/01-pre-audit-planning.md` first when starting from scratch.
4. Use the numbered reference files based on the exact bank-audit area.
5. Use the checklist and template assets when the user needs working material.
6. Use the scripts only as helper tools for calculation or structured working support; never treat their output as final without review.

## Good use cases

- NPA classification review
- IRACP provisioning working
- branch advances audit
- LFAR preparation
- MOC working
- RBI compliance checklists
- statutory branch and statutory central audit support

## Boundaries

- Do not issue final audit opinions autonomously.
- Do not claim regulatory compliance without human CA review.
- Keep all outputs review-oriented and evidence-oriented.
